Web Accessibility
The WAI initiative
The World Wide Web Consortium (W3C) is an international public-interest non-profit organization where Member organizations, a full-time staff, and the public work together to develop Web standards. Web Accessibility Initiative (WAI) is a key aspect of W3C’s work for the benefit of humanity.

W3C’s Web Accessibility Initiative (WAI) develops strategies, standards, and supporting resources to make the web accessible to people with disabilities.

The W3C vision is that the Web and related technologies are accessible so that all people with disabilities around the globe can participate equally in the digital world. That includes:

  • Accessibility standards/guidelines are harmonized globally.
  • All technical standards and specifications support accessibility.
  • Tools that create and deliver digital content support accessibility.
  • Everyone who creates or manages digital products has relevant accessibility knowledge.
  • New technologies, digital products, and content addresses accessibility from inception (“born accessible”).

To contribute to realizing the vision, W3C addresses the following areas:

  • Specifications, standards, and technologies support accessibility.
  • International accessibility standards.
  • Support understanding, adopting, and implementing accessibility.
  • Accessibility research.
Four tenets of web accessibility

Web content should be perceivable by at least one sense.

Web content should be operable, using a keyboard, mouse, voice, etc.

Web content should be understandable, it should make sense to the users.

Web content should be robust; it should use web standards compatible with different browsers, currently and in the foreseeable future.
WCAG-EM
W3C 's Website Accessibility Conformance Evaluation Methodology (WCAG-EM) is an approach for determining conformance to Web Content Accessibility Guidelines (WCAG).

WCAG-EM is for anyone who wants a common procedure for auditing websites. It is for direct use by internal evaluators, external auditors, benchmarkers, and researchers. Additionally, WCAG-EM can be referred to by managers, procurers, policy makers, regulators, and others.
Helpful Tip

When developing or redesigning a website or web application, evaluate accessibility early and throughout the development process to identify accessibility problems early, when it is easier to address them.

There are evaluation tools that help with evaluation. However, no tool alone can determine if a site meets accessibility standards. Knowledgeable human evaluation is required to determine if a site is accessible.

ARIA
Accessible Rich Internet Applications (ARIA) is a set of roles and attributes that define ways to make web content and web applications (especially those developed with JavaScript) more accessible to people with disabilities.

It supplements HTML so that interactions and widgets commonly used in applications can be passed to assistive technologies when there is not otherwise a mechanism. For example, ARIA enables accessible JavaScript widgets, form hints and error messages, live content updates, and more.
